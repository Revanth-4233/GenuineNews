🏗️ 1. Project Overview & Architecture

GenuineNews is a web-based AI system that detects whether a news article is TRUE, FALSE, PARTIALLY TRUE, or MISLEADING using a combination of:

  • BERT Sentence Embeddings — converts text into 384-dimensional vectors
  • Firefly Algorithm — bio-inspired feature selection to pick the best features
  • Multi-class SVM (MSVM) — Support Vector Machine with One-vs-Rest strategy
  • Web Evidence Search — cross-verifies with Google (SerpAPI), NewsAPI, and GDELT
User Input→ Text Cleaning→ BERT Embed→ PCA Reduce→ Firefly Select→ SVM Predict→ Evidence Verify→ Final Verdict

Registration Flow (Step by Step)

  1. User fills in username, password, confirm password, email on Register.html
  2. Client-side JavaScript validates passwords match before form submission
  3. Server-side RegisterAction() view receives the POST data
  4. Server checks: username not empty, passwords match, username not already taken
  5. Password is hashed using SHA-256: hashlib.sha256(password.encode()).hexdigest()
  6. New UserProfile record is created in SQLite database
  7. User is redirected to login page with success message

Login Flow

  1. User enters username and password on UserLogin.html
  2. UserLoginAction() hashes the entered password with SHA-256
  3. Queries database: UserProfile.objects.get(username=username, password=hashed)
  4. If match found → stores username in Django session → redirects to Dashboard
  5. If no match → shows "Invalid username or password" error

Session Management

Django's built-in session framework (django.contrib.sessions) stores sessions in the database. The session engine is configured as django.contrib.sessions.backends.db, meaning session data is stored in a django_session table in SQLite.

🤖 7. ML Pipeline — BERT + Firefly-MSVM

Step 1: Text Cleaning (clean_social_media_text())

Removes URLs, @mentions, RT markers, special characters. Lemmatizes words using NLTK's WordNetLemmatizer. Removes stopwords and short words.

Step 2: BERT Embeddings (BERTExtractor)

Uses all-MiniLM-L6-v2 model from sentence-transformers. Converts each text into a 384-dimensional vector that captures semantic meaning. Falls back to TF-IDF if sentence-transformers is unavailable.

Step 3: PCA Dimensionality Reduction

Reduces 384 features to ~100 components keeping 95% variance. Uses sklearn.decomposition.PCA with full SVD solver.

Step 4: Firefly Feature Selection (FireflyMSVM)

Bio-inspired algorithm with 12 fireflies and 5 iterations. Each firefly represents a binary feature mask. Fitness = cross-validated accuracy of SVM on selected features. Fireflies move toward brighter (better) solutions. Final result: only the most discriminative features are kept.

Step 5: Multi-class SVM Training

SVC with RBF kernel, C=200, balanced class weights, OvR strategy. 80/20 train-test split with stratification. Outputs: accuracy, precision, recall, F1-score, confusion matrix.

Step 6: Confidence Threshold

If prediction confidence < 50% → returns UNCERTAIN instead of potentially wrong label. This prevents unreliable predictions from being shown as definitive.

Model Persistence

Entire pipeline (BERT extractor, scaler, PCA, firefly, SVM, label encoder, metrics) is saved as model/detector.pkl using Python's pickle. Lazy-loads into RAM only when first prediction is needed — saves memory on free Render tier.

🔍 8. Evidence Verification System

After AI prediction, the system cross-verifies the claim using 3 APIs simultaneously:

APIPurposeLimit
SerpAPI (Google)Google search for the claim100 searches/month (free)
NewsAPISearch recent news articles100 requests/day (free)
GDELTGlobal news article databaseUnlimited (free, no key needed)

Verification Logic

  • Official Info (chairman, principal claims) → checks ONLY official .edu.in/.ac.in websites
  • College Events (fest, seminar) → checks ALL sources
  • General News → checks ALL sources, prioritizes trusted outlets (NDTV, TheHindu, BBC, Reuters, etc.)

Final Verdicts

CONFIRMED = found on trusted/official sources → REAL NEWS
CONTRADICTED = claim details don't match any source → FAKE NEWS
UNVERIFIED = not enough evidence found → relies on AI prediction
PARTIALLY-CONFIRMED = some details match, some don't

🚀 10. Deployment Configuration (Render)

  • Procfile: web: gunicorn Genuine.wsgi --workers 1 --threads 1 — single worker to stay within free memory limits
  • build.sh: Installs dependencies, downloads trained model from Google Drive using gdown, runs collectstatic and migrate
  • WhiteNoise: Serves static files directly from Django — no need for Nginx or S3
  • Lazy Loading: Model is NOT loaded at startup — only loaded when first prediction is requested, preventing OOM crashes
  • Dataset on Render: The Dataset/ folder (~430MB) is too large for Git, so it is NOT deployed. On Render, the "Load Dataset" page shows 420 sample records (built-in fallback). However, the pre-trained model (detector.pkl) was trained locally with the full 20,000 balanced records and is downloaded via build.sh — so predictions work perfectly with the same accuracy as local
  • Environment Variables: SECRET_KEY, DEBUG, ALLOWED_HOSTS, CSRF_TRUSTED_ORIGINS, API keys

▶️ 11. How to Run the Project

Double-click run_windows.bat — installs dependencies, migrates database, starts server at http://127.0.0.1:8000

Option 2: Manual Steps

  1. pip install -r requirements.txt
  2. python manage.py makemigrations
  3. python manage.py migrate
  4. python manage.py runserver
  5. Open browser → http://127.0.0.1:8000
